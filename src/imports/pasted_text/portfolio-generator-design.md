PROMPT START

Design a high-quality SaaS web application called “Portfolio Generator” that allows users to create, customize, and publish personal portfolio websites.

The design must feel like a real production-grade product similar in quality to Vercel, Linear, Framer, and Notion.

The UI should be clean, symmetrical, well-balanced, and visually refined, but must avoid looking like a generic AI-generated startup template.

⸻

Critical Design Direction

The design must feel intentionally crafted, not auto-generated.

Avoid:

* Generic startup landing page structures repeated across AI tools
* Overused SaaS template patterns found in UI kits
* Random or overly decorative gradients
* Excessive icon usage without functional purpose
* Overly “stock” hero layouts with cliché messaging blocks

Instead:

* Use a strong and consistent design system
* Maintain precise alignment, spacing, and grid structure
* Ensure visual hierarchy feels engineered, not templated
* Every section should feel purpose-built for this product specifically

⸻

Overall Visual Style

* Modern SaaS product UI
* Clean, structured, and balanced layout
* Strong grid system (12-column layout for web pages)
* Consistent spacing scale (4px or 8px system)
* Subtle elevation and layering
* Minimal but premium aesthetic

Think:

* Linear level UI precision
* Vercel dashboard clarity
* Notion structural simplicity

⸻

Typography System

* Font: Inter or SF Pro
* Strict hierarchy system:
    * H1: bold, tight spacing, high clarity
    * H2: medium weight, structured
    * Body: clean, readable, neutral tone
* Avoid overly large or decorative hero typography
* Maintain consistent vertical rhythm across pages

⸻

Color System

Light Mode

* Soft off-white background (not pure #FFFFFF)
* Neutral gray UI surfaces for layering
* One primary accent color (blue or violet preferred)
* Subtle borders for separation, not heavy shadows

Dark Mode

* Deep neutral background (true dark, not bluish neon)
* Layered surface hierarchy using slightly lighter dark tones
* Accent color remains consistent and controlled
* Avoid glowing or neon effects

⸻

Layout Principles

* Use strict grid alignment across all pages
* Ensure perfect visual balance and spacing consistency
* Do not overcrowd sections
* Maintain predictable rhythm in spacing and layout
* Use whitespace intentionally, not excessively

⸻

Application Structure

Design a full SaaS system with the following pages:

⸻

1. Landing Page (Product-Focused SaaS)

Structure:

* Hero section:
    * Clear headline: “Build and publish your portfolio in minutes”
    * Subheadline focused on speed and simplicity
    * One primary CTA and one secondary CTA
* Product preview section:
    * Real UI mock of dashboard/editor
    * Not illustration-style, but realistic interface preview
* Feature section:
    * 3–5 core features only
    * Focus on value, not marketing exaggeration
* Template preview section:
    * Clean horizontal or grid layout
    * Show actual template differences clearly
* Final CTA section

Design should feel like a real shipped SaaS homepage, not a marketing template.

⸻

2. Authentication Pages

* Clean login and signup screens
* Centered layout with strong hierarchy
* Minimal distractions
* Proper form spacing and alignment

⸻

3. Main Dashboard

Design a structured SaaS dashboard:

Layout:

* Left sidebar navigation (fixed, clean, consistent spacing)
* Top bar minimal (user, publish status, settings)

Sidebar:

* Dashboard
* Portfolios
* Templates
* Settings

Main content:

* Portfolio overview in structured list format (not overly card-heavy)
* Clear status indicators (draft, published)
* Primary CTA: “Create New Portfolio”
* Secondary actions subtle

⸻

4. Portfolio Editor (Core System)

This is the most important screen.

Layout: 3-column system

LEFT PANEL:

* Structured form inputs
* Collapsible sections:
    * Profile
    * About
    * Projects
    * Experience
    * Skills
* Clean labeled inputs, consistent spacing

CENTER PANEL:

* Live preview of portfolio
* Browser frame container (realistic, minimal styling)

RIGHT PANEL:

* Template selector (clean thumbnail cards)
* Theme controls:
    * Color picker
    * Font selector
    * Dark/light toggle
* Section ordering controls

Everything must feel highly usable, balanced, and production-ready.

⸻

5. Templates Page

* Clean grid layout
* 3–5 carefully designed templates
* Each template card includes:
    * Name
    * Short description
    * Preview
    * “Use Template” button

Templates:

* Professional
* Developer
* Creative
* Minimal
* Executive

⸻

6. Public Portfolio Page

Design a real production output page:

* Clean, structured layout
* Strong visual hierarchy
* Proper spacing rhythm
* Sections:
    * Hero (Name, Title)
    * About
    * Projects (grid with consistent alignment)
    * Experience (timeline layout)
    * Skills (tag system)
    * Contact section

Must feel like a real published website, not a mock UI.

⸻

Component Design Language

* Buttons: consistent radius, subtle hover states
* Inputs: clean borders, no heavy styling
* Cards: balanced padding, subtle border, no excessive shadows
* Tables/lists: preferred over unnecessary card grids in dashboard
* Modals: simple overlay, focused content only

⸻

UX Principles

* Everything is predictable and structured
* User understands flow immediately: Create → Edit → Publish
* No unnecessary complexity in navigation
* Strong emphasis on clarity and usability
* Designed for freelancers, developers, and professionals

⸻

Output Requirement

Generate a complete SaaS UI system with strong visual consistency, premium quality design, and structured layout across all pages.

The final result should look like a real production SaaS product that has been carefully designed and shipped, not a generic or AI-generated template.
